로드맵에는 현재 코드 상태(이미 완료된 Task는 `[x]` 체크박스, 미완료는 `[ ]`)를 반영하고, 각 Phase의 상태(완료/진행 중/미시작)를 명시한다.

**Why:** git status에 이미 초기 파일들이 존재하므로, "Phase 1 전체를 새로 만들어야 한다"는 혼선을 방지하기 위해 현재 구현 상태를 로드맵에 직접 반영해달라고 요청.

**How to apply:**
- API 연동/비즈니스 로직 Task에는 Playwright MCP 테스트 체크리스트 필수 포함
- "알려진 기술 부채" 섹션을 부록에 포함하여 우선순위별로 정리
- Task 명명: `Task XXX: [동사] [대상] [목적]` 형식 유지
- 각 위험 요소는 상황/완화 2단 구조로 작성
