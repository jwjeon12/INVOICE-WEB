# 노션 기반 견적서 웹 조회 & PDF 다운로드 MVP PRD

## 🎯 핵심 정보

**목적**: Notion에 입력된 견적서 데이터를 클라이언트가 웹으로 확인하고 PDF로 다운로드 받을 수 있는 플랫폼을 구축하여 견적서 공유의 번거로움을 제거한다.

**사용자**: 프리랜서/소상공인(발급자)과 이들의 클라이언트(수령자)

---

## 🚶 사용자 여정

```
[Admin: 노션 데이터베이스에 견적서 작성]
        ↓ 견적서 작성 완료
[시스템: 고유 공유 토큰 자동 생성]
        ↓
[Admin: 웹 인터페이스에서 공유 URL 복사]
        ↓ "공유 링크 복사" 클릭
[Admin: 이메일/메시지로 클라이언트에게 URL 전달]
        ↓
[Client: 받은 링크 클릭 → 웹에서 견적서 열기]
        ↓ 토큰 검증 ✓
[Client: 견적서 내용 확인]
        ↓
[Client 선택: "PDF 다운로드" 버튼 클릭]
        ↓ PDF 생성
[Client: 견적서 PDF 다운로드 완료]
        
[Client 선택: 링크 만료]
        ↓
[System: "링크 만료" 안내 페이지 표시]
```

## ⚡ 기능 명세

### 1. MVP 핵심 기능

| ID | 기능명 | 설명 | MVP 필수 이유 | 관련 페이지 |
|----|--------|------|-------------|------------|
| **F001** | Notion 데이터 연동 | `@notionhq/client` SDK로 Notion 데이터베이스에서 견적서 데이터 실시간 조회 | 핵심 데이터 소스로 Notion 활용 필수 | 견적서 조회 페이지 |
| **F002** | 공개 견적서 조회 | 고유 토큰(Notion 페이지 ID)을 URL 경로에 포함하여 인증 없이 견적서 공개 표시 | 클라이언트가 쉽게 접근 가능해야 함 | 견적서 조회 페이지 |
| **F003** | PDF 생성 및 다운로드 | 견적서 UI를 PDF로 변환하여 클라이언트가 다운로드 가능 (React PDF 또는 Puppeteer) | 오프라인 저장 및 인쇄 필요 | 견적서 조회 페이지 |
| **F004** | 유효성 검사 | 존재하지 않는/만료된 링크 접근 시 안내 메시지 표시 | 사용자 경험 개선 및 오류 방지 | 오류 페이지 |

### 2. MVP 이후 기능 (제외)

- 클라이언트 서명 (Signature) 기능
- 견적서 상태 추적 (수락/거절)
- 재무 이메일 템플릿 설정
- 다국어 지원
- 고급 필터링 및 검색
- 견적서 서식 커스터마이징 (테마/로고)

## 📄 페이지별 상세 기능

### 1. 견적서 조회 페이지

> **구현 기능:** `F001`, `F002`, `F003` | **경로:** `/quote/[token]` | **인증:** 불필요 (공개)

| 항목 | 내용 |
|------|------|
| **역할** | 클라이언트가 견적서 내용을 조회하고 PDF 다운로드하는 메인 페이지 |
| **진입 경로** | 관리자가 공유한 고유 링크(Notion 페이지 ID 포함)를 클라이언트가 브라우저에서 클릭 |
| **사용자 행동** | 1) 링크 열기 → 2) 견적서 내용 확인 → 3) PDF 다운로드 또는 페이지 닫기 |
| **주요 기능** | • Notion에서 실시간 견적서 데이터 로드<br>• 견적서 제목, 클라이언트명, 발행일, 만료일 표시<br>• 견적 항목 리스트 표시 (상품/서비스명, 수량, 금액)<br>• 합계 금액 강조 표시<br>• **"PDF 다운로드"** 버튼 (브라우저 다운로드 폴더에 저장)<br>• 반응형 디자인 (모바일/태블릿/데스크톱) |
| **다음 이동** | • 성공 → PDF 다운로드 또는 페이지 닫기<br>• 실패 (토큰 유효하지 않음) → 오류 페이지 리디렉션 |

### 2. 오류 페이지

> **구현 기능:** `F004` | **경로:** `/quote/error` 또는 `/not-found` | **인증:** 불필요

| 항목 | 내용 |
|------|------|
| **역할** | 존재하지 않거나 만료된 견적서 링크 접근 시 안내 메시지 제공 |
| **진입 경로** | 유효하지 않은 토큰으로 `/quote/[invalid-token]` 접근 시 자동 리디렉션 |
| **사용자 행동** | 오류 메시지 확인 → 링크 재요청 또는 발급자 연락 |
| **주요 기능** | • "요청하신 견적서를 찾을 수 없습니다" 메시지<br>• "링크가 만료되었거나 잘못된 것 같습니다" 설명<br>• "발급자에게 다시 요청하세요" 안내<br>• 홈 또는 이메일 문의 버튼 (선택) |
| **다음 이동** | • 홈으로 돌아가기 또는 발급자 연락 |

## 🗄️ 데이터 모델

### Notion 데이터베이스 스키마 (관리자가 Notion에서 관리)

| 필드 | 설명 | 타입 |
|------|------|------|
| **ID** | Notion 페이지 고유 식별자 (공개 공유 토큰) | UUID (Notion 내부) |
| **제목** | 견적서 제목 (예: "프로젝트명 견적서") | Title (Notion 기본) |
| **클라이언트명** | 견적 수령 클라이언트 이름 | Text |
| **견적 항목** | 서비스/상품 목록 (상품명, 수량, 금액) | Database Relation 또는 Rich Text |
| **합계금액** | 총 견적 금액 (자동 계산 또는 수동입력) | Number |
| **발행일** | 견적서 발행 날짜 | Date |
| **만료일** | 견적서 유효 기간 만료 날짜 | Date |
| **상태** | 견적서 상태 (초안, 발송, 수락, 거절) | Select: draft \| sent \| accepted \| declined |
| **설명/노트** | 추가 설명 또는 약관 | Rich Text |

### 항목 테이블 (선택사항: Relation 사용 시)

| 필드 | 설명 | 타입 |
|------|------|------|
| **상품명** | 서비스 또는 상품 이름 | Text |
| **설명** | 상세 설명 | Text |
| **수량** | 수량 | Number |
| **단가** | 단위당 가격 | Number |
| **금액** | 수량 × 단가 (자동 계산) | Formula |

## 🛠️ 기술 스택 (최신 버전)

### 🎨 프론트엔드 프레임워크

- **Next.js 16** (App Router, Dynamic Route `[token]`) - React 풀스택 프레임워크
- **React 19** - UI 라이브러리 (최신 동시성 기능)
- **TypeScript 5.6+** - 타입 안전성 보장

### 🎨 스타일링 & UI

- **Tailwind CSS v4** (`@tailwindcss/postcss`) - 유틸리티 CSS 프레임워크
- **shadcn/ui** - 고품질 React 컴포넌트 라이브러리
- **Lucide React** - 아이콘 라이브러리 (다운로드, 공유 등)

### 📝 폼 & 데이터 처리

- **Zod** - 데이터 검증 (Notion 응답 검증)

### 🗄️ 백엔드 & 데이터

- **Notion API** (`@notionhq/client`) - Notion 데이터베이스 조회
- **환경변수** (`NOTION_API_KEY`, `NOTION_DATABASE_ID`) - API 키 관리

### 📄 PDF 생성

- **@react-pdf/renderer** - 서버사이드 React 컴포넌트 → PDF 변환
  - 또는 **Puppeteer** - HTML → PDF 렌더링 (대안)

### 🚀 배포 & 호스팅

- **Vercel** - Next.js 15+ 최적화 배포 플랫폼
- **Edge Functions** (선택사항) - 토큰 유효성 검사 고속화

### 📦 패키지 관리

- **npm** - 의존성 관리

## 📐 구현 요약

### 필요한 패키지 설치

```bash
npm install @notionhq/client @react-pdf/renderer zod
```

### 필수 환경변수 (.env.local)

```
NOTION_API_KEY=secret_xxxxx
NOTION_DATABASE_ID=xxxxx
```

### 핵심 파일 구조

```
src/
├── app/
│   ├── quote/
│   │   └── [token]/
│   │       └── page.tsx          # 견적서 조회 페이지
│   └── not-found.tsx             # 오류 페이지
├── lib/
│   ├── notion.ts                 # Notion 클라이언트 초기화
│   └── quote.ts                  # 견적서 데이터 조회 함수
└── components/
    ├── QuoteViewer.tsx           # 견적서 표시 컴포넌트
    ├── PDFDownloadButton.tsx      # PDF 다운로드 버튼
    └── ui/                        # shadcn/ui 컴포넌트
```

### 개발 명령어

```bash
npm run dev              # 개발 서버 시작
npm run build            # 프로덕션 빌드
npm run lint             # 코드 린트
```
